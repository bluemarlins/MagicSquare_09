# create-pr

현재 브랜치의 변경을 스테이징하고, 의미 있는 커밋 메시지로 커밋한 뒤, `origin`에 푸시하고 **GitHub에 Pull Request**를 연다.

## 반드시 할 일

1. 프로젝트 스킬 **`.cursor/skills/create-pr/SKILL.md`**의 단계를 따른다. (본문 템플릿: `.cursor/skills/create-pr/pr-body-template.md`)
2. 터미널에서 직접 `git` / `gh` 명령을 실행한다. 사용자에게 “직접 실행하세요”로 넘기지 않는다.
3. PR 저장소: **bluemarlins/MagicSquare_09**, base 브랜치는 원격 기본 브랜치(보통 `main`)로 맞춘다.

## PR 본문에 포함할 정보

리뷰어를 위해 다음을 **마크다운**으로 채운다.

- **배경:** 왜 이 수정이 필요한지
- **변경 유형:** 신규 구현, 테스트 추가/변경, 버그 수정, 리팩터링, 문서, 설정/CI, 기타 중 해당하는 것
- **수정 요약:** 변경 범위·동작 영향을 불릿으로
- **리뷰 시 포인트:** 집중해서 봐야 할 부분, 리스크, 엣지 케이스
- **테스트:** 수행한 검증 또는 해당 없음과 이유

## 완료 후

생성된 **PR URL**을 사용자에게 알려준다.
